# rts
## 概述
一个为深度学习服务的任务调度系统，使用 Rust 编写。

## 设计目标
- 🚀高性能：使用 Rust 提供的高性能特性，确保任务调度的高效性和低延迟。
- 监控性：提供任务执行的状态和日志记录功能。
- 易用性：提供简单易用的 API 和命令行工具。
- 条件编排：支持根据任务状态进行条件编排，允许用户定义复杂的任务依赖关系和执行逻辑。
- [ ] 显卡管理：提供对显卡资源的管理和调度，确保深度学习任务能够高效利用 GPU 资源。

## 与 ts 的对比
[ts](https://github.com/justanhduc/task-spooler) 是我非常喜欢的一个任务调度系统，它的设计理念和功能对我有很大的启发。然而它也存在许多问题：
- 有时任务无法被删除，只能强行 kill ts 服务器进程。
- 默认情况下获取“上一次任务”的信息，但经常获取到某个更早的任务信息。
- 运行槽位的数量最小只能是 1，无法置零以暂停所有新任务分配。
- 深度学习任务分配显卡时总是随机分配，而用户通常希望有一个优先级顺序。
- 无法运行时指定所监控的显卡。
- 一个任务会绑定一张显卡，无法让多个任务按实际显存复用一张显卡。
- 所发布的二进制包已过时，重新编译 ts 需要安装 C++ 环境，且编译过程中还存在诸多问题。

最后，ts 项目已经很久没有更新了。虽然我非常喜欢 ts，但我决定自己实现一个新的任务调度系统，尽可能兼容 ts 的接口，同时解决上述问题。
